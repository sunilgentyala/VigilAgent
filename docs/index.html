      None of them audit content for the failure modes specific to AI-agent-authored code.

Three detectors, purpose-built for agent output

    

      

        
📦

        
Package Hallucination Guard

        

          Extracts newly added dependencies from package.json,
          requirements.txt, and Cargo.toml diffs and
          checks each one against the live npm, PyPI, and crates.io registries.
          A 404 means the agent likely invented a package name or got
          typosquatted, flagged HIGH severity. Network failures are
          reported as unverified, never as a false vulnerability.
        

        

          npmPyPIcrates.io
        

      

      

        
🔐

        
Prompt Injection Auditor

        

          Scans added comments and string literals for directives aimed at an
          AI agent: "ignore previous instructions," "as an AI you must...,"
          replacement system prompts, instructions to conceal actions from a
          reviewer, secret-exfiltration directives, zero-width steganography
          characters, and suspicious base64 blobs near injection keywords.
        

        

          indirect injectionsteganographyjailbreak patterns
        

      

      

        
⚠

        
AI Defect Heuristics

        

          Detects the security-bearing defects that show up disproportionately
          in agent-authored diffs: empty or comment-only catch blocks that
          swallow exceptions, hardcoded placeholder credentials, and naive
          cryptography: Math.random() for tokens, MD5,
          DES/3DES, AES-ECB.
        

        

          swallowed exceptionsplaceholder credsnaive crypto

Quickstart

    

      

        
Install

        
npm install -g vigilagent

      

      

        
Audit uncommitted changes

        
git diff | vigilagent

      

      

        
Audit a patch file

        
vigilagent ./agent-change.patch

      

      

        
CI security gate

        
git diff origin/main...HEAD \
  | vigilagent --fail-on-vuln

      

      

        
Markdown report

        
git diff main...feature \
  | vigilagent --markdown > report.md

      

      

        
JSON for tooling

        
git diff | vigilagent --json

      A dependency-free unified diff parser feeds structured per-file hunks to
      three independent detection modules running in parallel, whose findings
      are merged into a single report and rendered in your chosen format, with
      an optional --fail-on-vuln exit-code gate for CI.
